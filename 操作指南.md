# 此工程是如何一步步创建出来的？

> [!Important]
> 下列操作，基于您已经安装了 `uv` 命令行工具。
> [如何安装？](https://uv.doczh.com/getting-started/installation/#github-releases)

<br/>

## Python 环境管理

**查看可用 Python版本**
```shell
uv python list
```

<br/>


**选择要使用的 Python版本**

```shell
uv python pin 3.11.9
```

> [!Tip]
> 事实上，uv 不用刻意安装 python，只要你在当前项目指定或使用了某个版本的 python，uv 会自动查找并安装。
> (前提是版本真实存在)


<br/>


**创建 Python虚拟环境**

```shell
uv venv
```

<br/>


**激活 虚拟环境**

```shell
# windows(PowerShell)
.\.venv\Scripts\Activate.ps1

# 如果报错并提示无执行权限，请使用管理员权限打开powershell，然后执行如下命令
Set-ExecutionPolicy RemoteSigned

# macOS / Linux
source .venv/bin/activate
```

> [!Note]
> 激活后在终端命令的前面会多出一个前缀：(.venv) 或 (项目名称)。

> [!Important]
> 每次启动项目，都需要执行此命令。


<br/>


## UV 工程环境

**初始化 uv工程**
```shell
# 初始化到当前目录
uv init

# 初始化到一个新的目录中
uv init <dir-name>
```

<br/>


## dbt 工程环境

### 使用命令的方式（不推荐）
**安装 dbt-core**
```shell
uv pip install dbt-core
```

<br/>

**安装 数据源适配器**

```shell
uv pip install dbt-postgres
```

> [!Warning]
> 根据项目实际需求来，示例中使用的是 PostgreSQL


<br/>

### 通过配置文件的方式（推荐）
1.编辑 `pyproject.toml` 配置文件中的依赖项

```toml
[project]
name = "dbt"
version = "0.1.0"
description = "Add your description here"
readme = "README.md"
requires-python = ">=3.14.2"
dependencies = [
    "dbt-core"
]
```

2.更新并下载依赖

```shell
uv sync
```

> [!Tip]
> 下载龟速？临时指定软件下载源 `--index-url https://pypi.tuna.tsinghua.edu.cn/simple`


<br/>


## 验证 dbt
```shell
dbt --version
dbt debug
```

<br/>


## 初始化 dbt工程
```shell
# 初始化到一个新的目录中
dbt init <dir-name>
```

> [!Note]
> dbt 强制要求在一个空的目录中初始化 dbt工程。

<br/>

## dbt 命令

**加载种子**
```shell
dbt seed
```

## 查看编译后的模板化文件
```shell
# 指定文件路径时，推荐使用包名的形式
dbt compile --select example2.xxx > $null
```

## 直接运行模型(.sql)
```shell
dbt run --select example2.xxx
```
